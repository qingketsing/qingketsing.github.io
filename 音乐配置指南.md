# 音乐播放器配置指南

本博客系统支持为每篇文章单独配置背景音乐。只有配置了音乐的文章才会显示音乐播放器，避免不必要的流量消耗。

## 配置方法

### 方法一：直接配置
在Markdown文件的frontmatter中添加`music`配置：

```yaml
---
layout: "../../layouts/MarkdownPostLayout.astro"
title: "文章标题"
pubDate: 2025/1/1
auther: "作者"
type: "分类"
music:
  audioSrc: "/audio-file.mp3"    # 音频文件路径（必需）
  coverSrc: "/cover-image.jpg"   # 封面图片路径（可选）
  title: "歌曲名称"              # 歌曲标题（可选）
  artist: "艺术家"               # 艺术家名称（可选）
---
```

### 方法二：使用音乐库配置（推荐）
为了更好地管理音乐资源，我们提供了统一的音乐库配置。首先在`src/config/musicLibrary.js`中定义音乐：

```javascript
export const musicLibrary = {
  my_song: {
    audioSrc: "/songs/my-song.mp3",
    coverSrc: "/covers/my-cover.jpg",
    title: "我的歌曲",
    artist: "我的艺术家"
  }
};
```

然后在文章中引用：

```yaml
---
layout: "../../layouts/MarkdownPostLayout.astro"
title: "文章标题"
musicKey: "my_song"  # 引用音乐库中的配置
---
```

## 配置说明

### 必需参数
- `audioSrc`: 音频文件的路径，相对于public目录

### 可选参数
- `coverSrc`: 封面图片路径，如不设置将使用默认封面
- `title`: 歌曲标题，如不设置将使用默认标题
- `artist`: 艺术家名称，如不设置将使用默认艺术家

## 示例配置

### 完整配置示例
```yaml
music:
  audioSrc: "/songs/peaceful-music.mp3"
  coverSrc: "/covers/peaceful-cover.jpg"
  title: "宁静时光"
  artist: "Unknown Artist"
```

### 最简配置示例
```yaml
music:
  audioSrc: "/8_31.mp3"
```

### 不显示播放器
如果文章不需要音乐播放器，直接不添加`music`配置即可：

```yaml
---
layout: "../../layouts/MarkdownPostLayout.astro"
title: "无音乐文章"
pubDate: 2025/1/1
auther: "作者"
type: "分类"
# 没有music配置，不会显示播放器
---
```

## 优势

1. **按需加载**: 只有配置了音乐的文章才会加载播放器组件
2. **节省流量**: 避免在不需要音乐的页面加载音频文件
3. **灵活配置**: 每篇文章可以有独立的音乐设置
4. **优雅降级**: 没有配置音乐的文章正常显示，不受影响

## 注意事项

1. 音频文件需要放置在`public`目录下
2. 推荐使用`.mp3`格式以获得最佳兼容性
3. 封面图片建议尺寸为正方形，推荐400x400px
4. 文件路径以`/`开头，相对于网站根目录
